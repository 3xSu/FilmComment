# FilmComment 功能特性文档

## 项目概述

FilmComment 是一个基于 Spring Boot 开发的智能电影评论社区平台，集成了 AI 智能推荐、防剧透分区、实时互动等现代化功能。项目采用微服务架构设计，支持多端访问和智能交互。

## 核心功能模块

### 1. 电影信息管理
- **电影库管理**: 支持电影信息的增删改查，包含海报、简介、演员等详细信息
- **评分系统**: 用户可以对电影进行评分，系统自动计算平均分和统计信息
- **分类浏览**: 按类型、地区、年份等多种维度分类浏览电影
- **搜索功能**: 支持关键词搜索、条件筛选等高级搜索功能

### 2. 智能评论系统
- **评论发布**: 用户可以对电影发表评论，支持富文本编辑
- **防剧透分区**: 智能识别和隔离剧透内容，保护观影体验
- **评论互动**: 支持点赞、回复、收藏等互动功能
- **AI 评论分析**: 基于 Ollama 的 AI 模型对评论进行智能分析和总结

### 3. 用户社交功能
- **用户注册登录**: 完整的用户认证系统，支持 JWT Token
- **个人主页**: 用户个人资料、观影记录、评论历史展示
- **关注系统**: 用户可以关注其他用户，建立社交关系
- **消息通知**: 实时通知系统，支持 WebSocket 推送

### 4. AI 智能推荐系统
- **智能对话 Agent**: 基于 AI 的电影推荐助手，支持自然语言对话
- **个性化推荐**: 根据用户历史行为和偏好进行个性化电影推荐
- **RAG 推荐引擎**: 基于检索增强生成的智能推荐算法
- **偏好提取**: AI 自动分析用户对话历史，提取观影偏好

### 5. 管理后台功能
- **内容审核**: 管理员可以对用户发布的内容进行审核管理
- **数据统计**: 系统运行数据、用户行为数据统计分析
- **系统监控**: 系统性能监控和异常告警

## 技术架构特色

### 1. 多模块架构设计
项目采用模块化设计，主要包含：
- **FC-server**: 主服务模块，包含业务逻辑和 API 接口
- **FC-pojo**: 数据对象模块，包含实体类、VO、DTO 等
- **FC-common**: 公共模块，包含工具类、异常处理、配置等

### 2. AI 集成能力
- **Ollama 集成**: 本地 AI 模型集成，支持智能评论分析和总结
- **MCP 服务器**: 独立的电影 MCP 服务器，提供 TMDB 和豆瓣 API 接口
- **向量搜索**: 基于向量数据库的智能搜索和推荐

### 3. 实时交互功能
- **WebSocket 支持**: 实时消息推送和在线状态管理
- **Redis 缓存**: 高性能缓存系统，提升系统响应速度
- **布隆过滤器**: 高效的数据去重和查询优化

### 4. 安全与性能
- **JWT 认证**: 安全的用户认证和授权机制
- **文件安全验证**: 上传文件的安全检查和验证
- **自动清理任务**: 定时清理过期数据和优化系统性能

## AI Agent 功能详解

### 对话式电影推荐
- **自然语言交互**: 用户可以通过自然语言与 AI Agent 进行对话
- **上下文记忆**: Agent 能够记住对话历史，提供连贯的推荐体验
- **多轮对话**: 支持复杂的多轮对话场景，深入理解用户需求

### 智能工具调用
- **电影搜索工具**: 调用外部 API 获取最新电影信息
- **历史推荐工具**: 基于用户历史行为进行推荐
- **偏好分析工具**: 自动分析用户偏好，优化推荐策略

### 监控与统计
- **对话历史记录**: 完整的对话历史存储和检索
- **工具使用统计**: 监控 AI 工具的使用情况和性能
- **用户偏好分析**: 基于对话内容分析用户观影偏好

## 数据库设计

### 核心数据表
- **用户相关**: users, user_movie_relation, movie_rating
- **内容相关**: posts, comments, post_images, comment_images
- **社交相关**: post_likes, post_collections, notifications
- **AI 相关**: agent_conversation_history, agent_user_preferences, agent_tool_usage

### 智能数据视图
- **对话统计视图**: 用户对话行为统计分析
- **工具使用统计**: AI 工具使用情况监控
- **推荐效果分析**: 推荐算法的效果评估

## 部署与扩展

### 环境要求
- **Java 17+**: 运行环境要求
- **MySQL 8.0+**: 数据库服务
- **Redis 6.2+**: 缓存服务
- **Ollama**: AI 模型服务（可选）

### 扩展能力
- **微服务架构**: 支持服务拆分和独立部署
- **API 网关**: 可集成 API 网关进行流量管理
- **监控体系**: 支持集成 Prometheus、Grafana 等监控工具
